# 电机控制通讯协议文档 V1.1

## 1. 概述

本文档描述了电机测试通用平台通讯指令协议(V1.1)，用于控制多台设备上的多个电机。协议基于 AT 命令格式，通过串口通信实现对电机的控制、状态查询和配置。

## 2. 通讯基础

### 2.1 通信格式

所有命令和响应均采用 ASCII 文本格式，以 AT+ 开头，以回车换行符(0x0D 0x0A)结尾。

### 2.2 命令优先级

系统执行最后收到的指令。如果前面指令未执行完毕，则丢弃不执行。例如：电机正转 10 秒，正在运行中，又收到反转 5 秒，则马上执行反转 5 秒的指令。

### 2.3 命令格式

上位机发送：`AT+[命令名称]=[参数1],[参数2],...`
下位机响应：`AT+[命令名称]=[设备ID],[电机号],[其他参数]...`

**注意**：所有命令的前两个参数均为设备ID和电机号（唯一例外是 MotorSetDevID 命令，它仅需要设备ID参数）。

## 3. 命令详细说明

### 3.1 电机运转 (MotorRun)

控制指定电机以设定的速度和方向运行指定时间。

**命令格式**：
AT+MotorRun=[设备ID],[电机号],[方向],[PWM值],[运行时间(微秒)]

**参数说明**：
- 设备ID：设备的唯一标识号
- 电机号：1-4，表示控制哪个电机
- 方向：1表示CW正转，2表示CCW反转
- PWM值：0-10000，表示PWM占空比
- 运行时间：单位为微秒(us)，如果为0则持续运行

**响应格式**：
AT+MotorRun=[设备ID],[电机号],[使能状态],[PWM值],[运行时间(微秒)],[完成状态]

**完成状态**：0表示开始运行，1表示运行结束

**示例**：// 发送：控制设备1234的1号电机正转，PWM=5000，运行2秒 AT+MotorRun=1234,1,1,5000,2000000

// 响应（开始运行）： AT+MotorRun=1234,1,1,5000,2000000,0

// 2秒后响应（运行完成）： AT+MotorRun=1234,1,1,5000,2000000,1

### 3.2 电机停转 (MotorStop)

立即停止指定电机的运转。

**命令格式**：AT+MotorStop=[设备ID],[电机号]

**响应格式**：AT+MotorStop=[设备ID],[电机号],[使能状态]

**示例**：// 发送：停止设备1234的2号电机 AT+MotorStop=1234,2

// 响应： AT+MotorStop=1234,2,1

### 3.3 电机状态查询 (MotorStatusQuery)

查询指定电机的当前状态。

**命令格式**：AT+MotorStatusQuery=[设备ID],[电机号]

**响应格式**：AT+MotorStatusQuery=[设备ID],[电机号],[使能状态],[电压(mV)],[电流(mA)],[方向],[温度],[速度],[错误码],[硬件版本]

**参数说明**：
- 方向：0表示停止，1表示CW正转，2表示CCW反转
- 错误码：0表示无错误，1表示过压，2表示欠压，3表示过流（堵转），4表示其它错误

**示例**：// 发送：查询设备1234的1号电机状态 AT+MotorStatusQuery=1234,1

// 响应： AT+MotorStatusQuery=1234,1,1,12000,500,1,25,5000,0,V1.01.01

### 3.4 电机状态上报 (MotorStatusUpLoad)

设置指定电机定期上报状态的周期。

**命令格式**：AT+MotorStatusUpLoad=[设备ID],[电机号],[上报周期(ms)]

**响应格式**：AT+MotorStatusUpLoad=[设备ID],[电机号],[使能状态],[电压(mV)],[电流(mA)],[方向],[温度],[速度],[错误码],[硬件版本]

**示例**：
// 发送：设置设备1234的1号电机每500ms上报一次状态 AT+MotorStatusUpLoad=1234,1,500

// 定期响应： AT+MotorStatusUpLoad=1234,1,1,12000,500,1,25,5000,0,V1.01.01

### 3.5 电机状态上报停止 (MotorStatusUpLoadStop)

停止指定电机的状态上报。

**命令格式**：AT+MotorStatusUpLoadStop=[设备ID],[电机号]


**响应格式**：AT+MotorStatusUpLoadStop=[设备ID],[电机号],[使能状态]
**示例**：
// 发送：停止设备1234的1号电机状态上报 AT+MotorStatusUpLoadStop=1234,1

// 响应： AT+MotorStatusUpLoadStop=1234,1,1
### 3.6 设置设备ID (MotorSetDevID)

设置设备的唯一标识ID。

**命令格式**：
AT+MotorSetDevID=[设备ID]

**响应格式**：
AT+MotorSetDevID=[设备ID]


**示例**：
// 发送：设置设备ID为5678 AT+MotorSetDevID=5678

// 响应： AT+MotorSetDevID=5678

### 3.7 设置电机类型 (MotorSetType)

设置指定电机的类型。

**命令格式**：
AT+MotorSetType=[设备ID],[电机号],[类型]

**参数说明**：
- 类型：1=有刷直流电机, 2=无刷无感电机, 3=无刷有感电机, 4=步进电机

**响应格式**：
AT+MotorSetType=[设备ID],[电机号],[使能状态],[类型]

**示例**：
// 发送：设置设备1234的2号电机为有刷直流电机 AT+MotorSetType=1234,2,1

// 响应： AT+MotorSetType=1234,2,1,1

### 3.8 设置报警阈值 (MotorAlarmValue)

设置指定电机的报警阈值（过压、欠压、堵转）。

**命令格式**：
AT+MotorAlarmValue=[设备ID],[电机号],[过压值(mV)],[欠压值(mV)],[堵转电流(mA)]

**响应格式**：
AT+MotorAlarmValue=[设备ID],[电机号],[使能状态],[过压值(mV)],[欠压值(mV)],[堵转电流(mA)]


**示例**：
// 发送：设置设备1234的1号电机报警阈值 AT+MotorAlarmValue=1234,1,16000,9000,800

// 响应： AT+MotorAlarmValue=1234,1,1,16000,9000,800

### 3.9 电机循环运转 (MotorRunRepeat)

设置指定电机按照设定的参数循环正反转。

**命令格式**：
AT+MotorRunRepeat=[设备ID],[电机号],[PWM值],[正转时间(微秒)],[正转停止时间(微秒)],[反转时间(微秒)],[反转停止时间(微秒)],[循环总次数]

**参数说明**：
- 正转时间：电机正向旋转的持续时间
- 正转停止时间：正转结束后的停止时间
- 反转时间：电机反向旋转的持续时间
- 反转停止时间：反转结束后的停止时间
- 循环总次数：要执行的正反转循环次数，0表示无限循环

**响应格式**：
AT+MotorRunRepeat=[设备ID],[电机号],[使能状态],[PWM值],[正转时间],[正转停止时间],[反转时间],[反转停止时间],[循环总次数],[当前循环次数]

**参数说明**：
- 当前循环次数：0表示开始循环，>=1表示已完成的循环次数

**示例**：
// 发送：设置设备1234的1号电机循环运转，循环10次 AT+MotorRunRepeat=1234,1,5000,1000000,500000,1000000,500000,10

// 响应（开始循环）： AT+MotorRunRepeat=1234,1,1,5000,1000000,500000,1000000,500000,10,0

// 完成一个循环后响应： AT+MotorRunRepeat=1234,1,1,5000,1000000,500000,1000000,500000,10,1

// 完成所有循环后响应： AT+MotorRunRepeat=1234,1,1,5000,1000000,500000,1000000,500000,10,10

### 3.10 电机停止循环转 (MotorStopRepeat)

停止指定电机的循环运转。

**命令格式**：
AT+MotorStopRepeat=[设备ID],[电机号]

**响应格式**：
AT+MotorStopRepeat=[设备ID],[电机号],[使能状态],[完成循环次数]


**示例**：
// 发送：停止设备1234的1号电机循环运转 AT+MotorStopRepeat=1234,1

// 响应： AT+MotorStopRepeat=1234,1,1,3

## 3.13 电机自定义阶段运行 (MotorRunCustom)

设置电机按照指定的多个不同阶段循环运行。每个阶段可以设定不同的运行方向和PWM值。

**命令格式**：
AT+MotorRunCustom=[设备ID],[电机号],[总周期时间(ms)],[重复次数],[阶段数],[阶段1开始时间],[阶段1状态],[阶段1PWM],...

**参数说明**：
- 设备ID：设备的唯一标识号
- 电机号：1-4，表示控制哪个电机
- 总周期时间：一个完整周期的持续时间(毫秒)
- 重复次数：周期重复执行的次数(0表示无限循环)
- 阶段数：一个周期内包含的阶段数量
- 阶段X开始时间：该阶段开始的时间点(相对于周期开始，单位毫秒)
- 阶段X状态：该阶段的运行状态(0=停止，1=正转，2=反转)
- 阶段X PWM：该阶段的PWM值(0-10000)

**注意**：
- 阶段的开始时间必须递增排序
- 最后一个阶段的开始时间必须小于总周期时间
- 第一个阶段的开始时间通常为0
- 阶段数最多支持20个

**响应格式**：
AT+MotorRunCustom=[设备ID],[电机号],[使能状态],[总周期时间],[重复次数],[阶段数],[当前循环次数]

**示例**：
// 发送：控制设备1234的1号电机，50秒周期内5个不同运行阶段，重复10次 AT+MotorRunCustom=1234,1,50000,10,5,0,1,5000,12000,0,0,30000,2,5000,42000,1,5000,46000,2,5000

// 响应（开始执行）： AT+MotorRunCustom=1234,1,1,50000,10,5,0

// 完成一个周期后响应： AT+MotorRunCustom=1234,1,1,50000,10,5,1


## 3.14 停止电机自定义运行 (MotorStopCustom)

停止电机的自定义阶段运行。

**命令格式**：
AT+MotorStopCustom=[设备ID],[电机号]

Code

**响应格式**：
AT+MotorStopCustom=[设备ID],[电机号],[使能状态],[已完成循环次数]

Code

**示例**：
// 发送：停止设备1234的1号电机自定义运行 AT+MotorStopCustom=1234,1

// 响应： AT+MotorStopCustom=1234,1,1,3
